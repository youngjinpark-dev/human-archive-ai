# Human Archive AI -- Persona Store 기획서

> 작성일: 2026-04-07
> 작성자: 서기 (기획 회의 결과 종합)
> 참여자: 마케팅, 법률, 분석가, 비평가, 아이디어, 기획자, 서기

---

## 1. 프로젝트 개요

Human Archive AI의 Persona Store는 **전문가의 지식과 인사이트를 AI 어시스턴트로 구축하여 판매하는 마켓플레이스**이다. 전문가(셀러)가 자신의 경험, 저서, 강연 등을 기반으로 AI 페르소나를 만들고, 이를 필요로 하는 사용자(바이어)가 구매하여 대화할 수 있는 플랫폼을 제공한다.

### 핵심 가설

> "10-20개 질문 + YouTube 자막으로 만든 AI가 전문가 상담료를 낼 만한 가치를 제공하는가?"
> -- 비평가 제기, 전원 동의

이 가설의 검증이 프로젝트 성패의 핵심이며, Phase 0 무료 베타에서 우선적으로 검증한다.

## 2. 핵심 가치 제안

### 포지셔닝 (확정)

**"전문가가 직접 감수한 AI 어시스턴트"**

- 제안: 비평가, 지지: 마케팅/아이디어/법률/기획자 (전원 합의)
- "전문가 복제", "진짜 전문가" 등의 표현은 사용 금지
- 셀러가 주체적으로 참여하고 검증한 도구임을 강조
- 법적 리스크(과대광고, 인격권 침해) 최소화
- 레벨 시스템 도입 시(Phase 3) "전문가가 N시간 투자한 Lv.X 어시스턴트" 표현 가능

### 사용 금지 마케팅 문구

- "전문가 수준의 답변"
- "전문가를 AI로 복제"
- "정확한 조언" / "믿을 수 있는 답변"
- "AI 변호사" / "AI 의사" 등 직업 사칭 표현

### 권장 마케팅 문구

- "OO 전문가의 경험과 인사이트를 바탕으로 한 AI 대화"
- "전문가가 직접 구축한 AI 어시스턴트와 대화하세요"
- "OO 분야 20년 경력의 지식을 AI로 만나보세요"

## 3. 타겟 시장 및 고객

### 시장 규모 (분석가 추정)

| 지표 | 수치 |
|------|------|
| TAM (한국) | 2~3조원 |
| Y1 SOM | 1.3억 (아카이빙 50건 + 구독 30명) |
| Y1 제작비 수입 | 1~2.5억 |
| 3년 누적 SOM | 15~25억 |

### 초기 타겟 버티컬 (합의)

**IT/개발 + 비즈니스/크리에이티브** 중심으로 시작 (분석가 + 마케팅 합의)

- 규제 리스크가 낮은 분야 우선
- 고위험 분야(의료/법률/재무)는 MVP에서 **완전 제외** (화이트리스트 방식으로 허용 카테고리만 오픈)
  - 비평가 강력 주장, 마케팅/법률 동의. "타협의 여지 없음"
  - 법적 프레임워크 완성 후 단계적 개방

### 고객 세그먼트

**셀러 (전문가)**
- 마이크로 전문가 (팔로워 1K~10K) 중심 Founding Creators
- 전문성 기준 선발 (팔로워 수가 아닌 실력 기반)

**바이어**
- B2C: 전문 지식이 필요한 개인 사용자
- B2B: 기업 내부 교육/컨설팅 대체 수요 (Phase 2부터 본격 파일럿)

## 4. 주요 쟁점 및 합의 사항

### 쟁점 1: 수수료 구조

| 의견 제시자 | 주요 입장 |
|-------------|-----------|
| 마케팅 | 80:20 시작, 70:30이 업계 표준 (Udemy/App Store 비교) |
| 비평가 | 40% 수수료 높다는 초기 지적 → 80:20 시작에 조건부 동의 |
| 분석가 | 제작비+구독 분리, 35% 상한, 손익분기 분석 제공 |
| 아이디어 | 레벨 연동 차등 수수료 제안 → v2로 이연 동의 |
| 기획자 | 등급별 마일스톤 기반 전환 최종안 제시 |
| 법률 | 수수료 변경 시 30일 사전 고지 조항 필수 |

**최종 합의**: 등급별 차등 수수료, 마일스톤 기반 전환

| 등급 | 수수료 (셀러:플랫폼) | 전환 조건 |
|------|----------------------|-----------|
| Founding Creator (30명) | 85:15 | 영구 우대 |
| 일반 | 80:20 | 기본 |
| 성장 | 70:30 | 월 구독자 500명 달성 시 |
| 성숙 | 60:40 | 월 구독자 1,000명+ 달성 시 |

- 약관에 변경 조건 사전 명시 + 30일 사전 고지 + 이의제기 기간 (법률 권고)
- 제작비(200~500만원) + 구독 수수료 이중 수익 구조 (분석가 제안, Phase 2 구독 도입 시 적용)

### 쟁점 3: 음성인증 타이밍

**전원 합의**: MVP 제외, Phase 3에서 프리미엄 기능으로 도입

- 법률: 비공인 인증, 법적 효력 없음 + Deepfake 리스크

- 마케팅: 음성 출시를 마케팅 이벤트로 활용 제안
- MVP에서는 서면 동의 + 내부 검수로 대체

### 쟁점 6: B2B vs B2C 우선순위

| 의견 제시자 | 주요 입장 |
|-------------|-----------|
| 분석가 | B2B 파일럿 우선 (10개사 = B2C 100명 매출 효과) |

**최종 합의**: B2C 스토어 우선, B2B는 Phase 2에서 파일럿

## 5. 제품 기능 상세

### Phase 0: 무료 베타 (6주) -- 9개 API

| 기능 | 설명 |
|------|------|
| 스토어 리스팅 | CRUD 4개 + 검색/필터 1개 |
| 시식 (Trial) | 시식 세션 생성 + 대화 (2개 API) |
| 접근 제어 | 초대 코드 기반 접근 (2개 API) |

- Founding Creators 30명 온보딩 (풀서비스: 멀티소스 크롤링 + 인터뷰 진행)
- 2문답 무료 시식: 페르소나당 2문답, 사용자당 일 3개 페르소나 제한
- 비로그인 시식 허용 (browse-wrap 약관 동의, 대화 데이터 세션 종료 시 즉시 삭제)
- 시식 → "계속 대화하기" CTA → 결제 전환 플로우 구축

### Phase 1: 결제 MVP (+4주, 총 10주) -- +3개 API

| 기능 | 설명 |
|------|------|
| 결제 (Toss Payments) | 결제 요청 + 승인 + 웹훅 (3개 API) |

- 건당 과금 1,000~3,000원 (Phase 0 데이터로 최종 결정)
- 건당 5만원 이하 가격 상한 (에스크로 면제용 -- 법률 권고)
- 정산: 월 1회 수동 계좌이체 ("매월 말 마감, 익월 15일 이내" 약관 명시)

### Phase 2: 확장 (+4주) -- +6개 API

- 판매자 대시보드
- 자동 정산 시스템
- B2B 파일럿 (3~5개사, 기업 대시보드 + 세금계산서 + 멀티시트 라이선스)
- 페르소나 비교 (같은 질문을 2개 페르소나에게 보내고 응답 나란히 표시)
- 구독 모델 추가 검토

### Phase 3: 성숙 (+6주) -- +8개 API

- 리뷰/평점 시스템
- 구독 결제
- 음성인증 (프리미엄 기능)
- 레벨업 시스템 (게이미피케이션)
- 멀티소스 확대 (PDF, 논문 임포트)

### Quality Gate -- 페르소나 등록 최소 요건 (확정)

| # | 조건 | 기준 |
|---|------|------|
| 1 | 인터뷰 완료 | 최소 9개 핵심 질문 + 3개 시나리오 응답 |
| 2 | 지식 소스 | YouTube 자막 30분+ OR 문서 자료 5개+ |
| 3 | 내부 QA | 5개 표준 질문 테스트, 관련성 점수 3/5 이상 |
| 4 | 전문가 확인 | 셀러 본인이 시식 응답 검토 후 승인 |

- 셀러 승인 단계가 핵심 (비평가 강력 제안): "이 AI가 내 이름으로 나가도 된다"는 셀러의 명시적 동의
- 사후 품질 관리: 평점 2.5 미만 자동 비공개, 부정확 신고 3건 시 수정 요청

### 면책 문구 체계 (확정)

- **기본**: "AI가 생성한 응답이며 전문적 조언이 아닙니다" -- 모든 대화에 상시 노출
- **고위험 감지 시**: 인라인 추가 면책 삽입 (의료/법률/투자 관련 응답 감지)
- **무료 시식**: 무료여도 면책 필수 표시

## 6. 기술 아키텍처

### 기술 스택

- **프론트엔드**: Next.js (이미 `src/` 내 구현 존재)
- **백엔드/인증**: Supabase Auth + RLS
- **결제**: Toss Payments (Phase 1)
- **AI 엔진**: 기존 chat 엔진 활용 (`src/` 내 chat 모듈)
- **시식 비용**: Haiku 4.5 모델, 월 ~17만원 (MAU 1,000 기준)

### MVP API 구조 (Phase 0 + Phase 1 = 12개)

| 카테고리 | API 수 | 비고 |
|----------|--------|------|
| 스토어 리스팅 | 5개 | CRUD + 검색 |
| 결제 (Toss) | 3개 | 결제요청 + 승인 + 웹훅 |
| 접근 제어 | 2개 | 구매확인 + 페르소나 접근 |
| 시식 (Trial) | 2개 | 시식 세션 생성 + 대화 |

### 시식 비용 통제

- 페르소나당 2문답, 사용자당 일 3개 페르소나 제한
- Haiku 4.5 사용으로 토큰 비용 최소화
- 비로그인 시식 대화 데이터 세션 종료 시 즉시 삭제

### 인증 뱃지 구조 (확정)

뱃지 명칭은 "품질 인증"이 아닌 **"신원 확인(Identity Verified)"** + **"데이터 충실도(Data Completeness)"**로 설정 (품질 보증 오해 방지 -- 아이디어 제안, 법률 검토 반영)

| 레벨 | 내용 | 시기 |
|------|------|------|
| Lv.1 | 이메일 인증 | MVP |
| Lv.2 | 실명 인증 (NICE/KCB 대행, 직접 수집 금지) | MVP |
| Lv.3 | 전문가 뱃지 (자격증/경력 확인) | Phase 3+ (법적 프레임워크 완성 후) |
| Lv.3+ | 음성인증 추가 | Phase 3 |

## 7. 수익 모델

### 이중 수익 구조 (분석가 제안, 전원 동의)

**1. 아카이빙 제작비**
- 건당 200~500만원 (플랫폼 100% 수취)
- 서비스 용역 대가 (세금계산서 발행)
- Y1 목표: 50건 = 1~2.5억

**2. 이용 수수료**
- Phase 1: 건당 과금 1,000~3,000원
- Phase 2+: 구독 모델 추가 검토
- 수수료: 등급별 차등 (85:15 ~ 60:40)

### 핵심 재무 지표 (분석가 제공)

| 지표 | 수치 |
|------|------|
| 월 고정비 | ~2,000만원 |
| Break-even (80:20, 제작비 포함) | 월 구독 거래액 4,000만원 |
| 시식 → 유료 전환율 (예상) | 12~18% |
| Y1 SOM | 1.3억 (아카이빙 50건 + 구독 30명) |
| Y1 제작비 수입 | 1~2.5억 (운영비 충당 가능) |
| 3년 누적 SOM | 15~25억 |

### B2B 가격 (Phase 2)

- 페르소나당 월 50~200만원 (기업 규모별 차등)
- B2B 10개사 = B2C 100명 매출 효과

|---|------|
| 1 | 상세 라이선스 등급 (개인/기업/API) |
| 2 | AI 기반 자동 필터링 |
| 3 | 셀러 외부 홍보 가이드라인 |
| 4 | 해외 셀러 원천징수 |
| 5 | 에스크로 (건당 5만원 초과 시) |
| 6 | 음성 데이터 관련 동의 |

### 정산 구조 (확정)

- MVP: 월 1회 수동 계좌이체 (법적으로 가능)
- 약관에 "매월 말 마감, 익월 15일 이내 정산" 명시
- 원천징수: 개인 셀러 8.8%, 사업자 셀러 세금계산서 수수
- 판매자 50명 이하 시 세무사 월별 일괄 위탁으로 간소화

### 무료 시식 법적 요건 (확정)

- 비로그인 허용, browse-wrap 약관 동의 (버튼 클릭 = 동의, 약관 링크 명확 표시)
- 대화 데이터 세션 종료 시 즉시 삭제 → 별도 개인정보 동의 불필요
- 면책 문구 필수 표시 (무료 ≠ 면책 면제)

### 신고/삭제 프로세스 (확정)

- 정보통신망법 제44조의2 준수
- 신고 접수 → 3영업일 검토 → 임시조치 또는 기각
- Bronze 레벨 무단 페르소나 생성 방지 포함

### 셀러 승인의 법적 효과 (법률 확인)

- 3중 방어 구조: 셀러 승인(셀러 책임) + 시스템 안전장치(플랫폼 주의의무) + 자동 면책(소비자 고지)
- 단독 면책은 불가하나, 과실 경감 + 구상권 확보 + 통신판매중개자 면책 요건 충족에 기여

### 법률 자문 예산

- 초기 법률 자문 비용: 500만~1,000만원 (비평가 제안)
- 이용약관 + 개인정보처리방침 검토: 200~500만원 (분석가 추정)

## 9. 리스크 및 대응 전략

### 리스크 1: 페르소나 품질 부족

- **위험**: AI 응답이 전문가 수준에 미치지 못해 유료화 가치 미달
- **대응**: Phase 0 무료 베타에서 핵심 가설 우선 검증. Quality Gate 4개 조건 적용. 셀러 본인 승인 필수.
- **비평가 경고**: "이 가설이 No이면 결제 시스템을 만드는 것 자체가 낭비"

### 리스크 2: 오조언 법적 책임

- **위험**: AI 응답을 따른 사용자의 피해 발생 시 플랫폼 책임 소지
- **대응**: 고위험 카테고리 MVP 제외 + 3중 방어(셀러 승인/시스템 안전장치/면책 문구) + 진단/처방/법적판단/투자추천 시스템 프롬프트 거부

### 리스크 3: 셀러 확보 실패

- **위험**: 초기 전문가 모집 부진
- **대응**: Founding Creator 85:15 영구 우대 + 풀서비스 온보딩 + 마이크로 전문가(1K~10K 팔로워) 중심 접근

### 리스크 4: 무단 페르소나 생성

- **위험**: 타인이 본인 동의 없이 페르소나 등록
- **대응**: 신원 확인 뱃지 체계 + 신고/삭제 프로세스(3영업일 검토) + 정보통신망법 44조의2 준수

### 리스크 5: 개발 일정 지연

- **위험**: 현재 코드베이스에 테스트 0개, 에러 처리 미흡 (분석가 지적)
- **대응**: Phase 0을 무료 베타로 설정하여 결제 복잡도 제거 + 코드 품질 개선 시간 확보

### 리스크 6: 셀러 리텐션 저하

- **위험**: 셀러 참여도 하락으로 페르소나 품질 저하
- **대응**: 90일 미활동 셀러 페르소나 비공개 처리(Phase 1) + 셀러 월간 응답 검토율 KPI 추적

## 10. 로드맵

### 확정 로드맵 (기획자 최종안, 전원 합의)

| Phase | 기간 | 핵심 목표 | API | 매출 목표 |
|-------|------|-----------|-----|-----------|
| **0: 무료 베타** | 6주 | 스토어 리스팅 + 시식(2문답) + 초대코드 접근. Founding Creators 30명 온보딩. 핵심 가설 검증 | 9개 | 제작비 2,000만~5,000만 |
| **1: 결제 MVP** | +4주 (총 10주) | Toss 건당결제 + 웹훅 + 접근제어. 베타 데이터 기반 가격 확정 | +3개 | 건당 과금 시작 |
| **2: 확장** | +4주 (총 14주) | 판매자 대시보드 + 자동정산 + B2B 파일럿(3~5개사) + 페르소나 비교 + 구독 모델 검토 | +6개 | 월 구독 500만+ |
| **3: 성숙** | +6주 (총 20주) | 리뷰/평점 + 구독 결제 + 음성인증 + 레벨업 | +8개 | 월 구독 2,000만+ |

### GTM 순서 (비평가 + 마케팅 합의)

1. **Phase 0**: 마이크로 전문가(1K~10K 팔로워) 중심 Founding Creators 30명 모집
2. **Phase 1**: B2C 건당 과금 시작 + B2B 영업 병행 (3~5개사)
3. **Phase 2**: 품질 검증 완료 후 인플루언서 확대 + B2B 본격화

### Phase 0 온보딩 전략

- Founding Creators 30명: 전문성 기준 선발 (팔로워 수 아닌 실력 기반)
- 풀서비스 제공: 플랫폼 측에서 멀티소스 크롤링 + 인터뷰 진행
- Phase 2부터 셀프서비스 온보딩 도구 제공
